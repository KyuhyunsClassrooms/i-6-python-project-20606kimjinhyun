# 최종 보고서

## 1. 프로그램 주제와 목적

내 프로그램의 주제:동물의 종류,체온,증상 분석을 통한 진단

이 프로그램을 만든 이유:
동물병원에 가기 힘든 경우에 이 프로그램을 통해 빨리 동물의 상태를 진단할 수 있게하기 위해서

---

## 2. 문제 분해 및 모델링

### 입력
동물 이름  동물 종류  체온  증상

### 처리
체온 범위에 따라 상태 판단
증상과 체온 상태에 따라 위험 점수 계산

### 출력
이름 종류 체온 체온상태 감지된증상 위험점수 최종판정 추천행동

---

## 3. 사용한 2차원 리스트 설명

2차원 리스트 이름:pets

한 행의 의미:한 마리의 동물의 정보

각 열의 의미:동물 이름 동물 종류 체온 증상 문장

| 열 번호 | 의미 | 실제 예시 |
| 0 | 동물 이름 | "초코" |
| 1 | 동물 종류 | "강아지" |
| 2 | 체온 | 39.8 |
| 3 | 증상 문장 | "구토하고 무기력함" |


이 2차원 리스트를 사용한 이유:여러 개의 정보를 묶어서 효율적으로 저장하고 처리하기 위해서이다.


---

## 4. 함수 설명

| 함수 이름 | 역할 | 입력값 | 반환값 또는 출력 |
|check_temperature()|체온 판정|동물 종류,체온|체온상태,위험도|
| analyze_symptom()| 발견 증상과 위험 점수 측정| 증상 문장|증상리스트,위험 점수 합계  |
|judge_health(score)| 최종 건강 판정|죄종 위험 점수|위험 양호 주의 필요|
| recommend_action(result, level)|추천 행동 출력|체온 상태, 최종 건강 판정|

가장 중요한 함수 하나를 골라 자세히 설명하세요.

함수 이름:recommend_action

설명:recommend_action() 함수는 최종 건강 판정과 체온 상태를 입력받아, 동물의 상태에 맞는 보호자의 대처 방법을 문자열로 반환하는 함수이다.

## 5. 조건문과 반복문 활용

### 조건문 사용 부분

어떤 조건을 검사했는가?
체온이 정상범위에 들어가는가


### 반복문 사용 부분

사용자가 입력한 동물 수만큼 반복하여 여러 동물의 정보를 입력받기 위해 사용함.


### 조건문과 반복문이 함께 작동한 부분

반복문으로 모든 증상을 하나씩 검사하고, 조건문으로 입력 문장에 해당 증상이 포함되어 있는지 판단한다.

## 7. 테스트와 오류 수정

### 테스트 1: 정상 사례

입력:증상에 따른 위험 정수 계산 함수에 있는 증상이 들어간 문장 입력

입력:증상에 따른 위험 점수 계산 함수에 없는 증상이 들어간 문장 입력

## 9. 한계와 개선 방안

현재 프로그램의 한계:모든 증상을 아직 위험 점수로 계산하기 어렵고 위험 점수로 추천 행동이 너무 구체적이지 못하다
